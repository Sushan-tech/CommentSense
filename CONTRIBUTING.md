# Contributing to CommentSense

Thank you for your interest in contributing to CommentSense. This project welcomes students and developers who want to improve content moderation systems through open-source collaboration.

---

## Who Can Contribute

- Students
- Open-source contributors
- Developers interested in backend APIs or moderation systems

Basic knowledge of JavaScript is helpful but not mandatory.

---

## Ways to Contribute

You can contribute in several ways:

- Improve chat segregation logic
- Add new sentiment or moderation categories
- Enhance frontend UI
- Add support for additional languages
- Improve documentation
- Refactor existing code for clarity

---

## Contribution Workflow

1. Fork the repository on GitHub
2. Clone your fork to your local machine
3. Create a new branch for your feature or fix
4. Make your changes
5. Commit your changes with a clear message
6. Push your branch to your fork
7. Open a Pull Request to the main repository

---

## Branch Naming Convention

Use meaningful branch names, for example:

- `feature/add-language-support`
- `fix/api-response-structure`
- `docs/update-readme`

---

## Coding Guidelines

- Follow the existing project structure
- Write clean and readable JavaScript
- Comment complex logic
- Do not commit `node_modules`
- Do not include API keys or sensitive data

---

## Pull Request Guidelines

- Clearly describe what your change does
- Keep pull requests focused on a single feature or fix
- Reference related issues if applicable

---

## Code of Conduct

Be respectful and professional in all discussions and contributions. Harassment or inappropriate behavior will not be tolerated.
